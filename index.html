02 — Worked example

Hermes, adapted — the assistant that briefs me before I open the laptop.

Hermes started as Nous Research's open agent framework. I've adapted and tuned it into my own self-hosted platform — an Ubuntu VM, Tailscale-only, thinking with local models. First job: the morning briefing, on my desk by 06:00.

I didn't write the framework — I made it mine. Hundreds of hours of adaptation and fine-tuning, and it now does the heavy lifting in my day: six MCP servers, forty-plus tools, twenty-eight skills — briefings, Telegram nudges, voice via Whisper, a Streamlit dashboard. It expands every week, because it's infrastructure I live on, not a portfolio piece.

The adaptation isn't the point either. The decisions underneath it are — every choice on the right I can defend under questioning. And the hours spent inside someone else's agent architecture are exactly what's compressing my ramp on Claude Code now.

Live · in production Ollama — local LLM Tailscale-only LangGraph + CrewAI Vector memory MCP → Gmail / Calendar
Decisions I'd defend in an interview
Why local-first?
Privacy and cost. My inbox and calendar are the most sensitive data I own, and a daily 06:00 run on metered tokens punishes you. Local inference amortises to zero marginal cost.
Why Tailscale, not a public endpoint?
Zero inbound surface. A personal assistant doesn't need to be reachable from the internet. It needs to be reachable from me.
Why LangGraph for the briefing?
Observability. The flow is fixed and stateful — when it misfires at 06:00 I inspect each node's state instead of reconstructing agent chatter. CrewAI earns its place on looser, multi-role work.
Why a vector store at all?
Retrieval over my own notes is the difference between an assistant and a parrot. Without it the briefing knows today; with it, it knows what I've been working on for weeks.
